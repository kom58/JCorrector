Los comandos disponibles
 

Corrector utiliza los comandos para reconfigurarse, de forma automática, mientras se ejecuta. Con ellos podemos visualizar u ocultar botones y ventanas, cargar nuevas páginas o archivos, iniciar secuencias musicales o multimedia e, incluso, ejecutar otras aplicaciones.

La gran variedad de posibilidades que ofrecen hacen recomendable un uso prudente de los mismos ya que, mal utilizados, podrían llegar a bloquear el propio programa y obligarnos a una completa reinstalación.

Estos comandos siempre van asociados a una Ficha y pueden ser de dos tipos :
 

Comandos de inicio

Se ejecutarán una sola vez, de forma automática, cuando carguemos la Ficha. Si tenemos configurada una Ficha de arranque, se ejecutarán al abrir el programa.
 

Comandos de pregunta

Se ejecutarán cuando accedamos a la respuesta de esa pregunta. Por defecto también se ejecutan sólo una vez, pero el Comando <SiRep> nos permitirá repetir su ejecución.
     

Lista de Comandos disponibles
     
<SB: ... >   Visualiza los botones que aparecen en la lista
<NB: ... >   Oculta los botones que aparecen en la lista
    Lista de botones
  1 Botón Abrir nueva Ficha
  2 Botón Inicio
  3 Botón Para
  4 Botón Página anterior
  5 Botón Página siguiente
  6 Botón Recargar página
  7 Botón Contestar preguntas (Corrector)
  8 Botón Ayuda de la Ficha
  9 Botón Calculadora
  0 Botón Bloc de notas
     
    De esta forma, si escribimos :
   

<NB:23456>
    Estaremos ocultando los botones de navegación, desde Inicio hasta Recargar.
     
<SiPant>   Visualiza la pantalla de la Ficha
<NoPant>   Oculta la pantalla de la Ficha
Este Comando se debe usar con prudencia para evitar situaciones que puedan bloquear el propio programa.
     
<SiCrr>   Visualiza la ventana del Corrector
<NoCrr>  

Oculta la ventana del Corrector
No se debe utilizar conjuntamente con <NoPant> ya que dejaríamos el programa abierto pero no visible.
     
<End>   Fin del programa.
Este comando cierra el programa. Se puede utilizar tras abrir otro programa o mostrar la pantalla completa con algún archivo.
     
<SiRep>   Repite la ejecución del comando si se vuelve a visualiza el mismo número de la pregunta. (Sólo en los Comandos de pregunta)
<NoRep>   No repite la ejecución del comando. Éste sólo se ejecutará una vez. Opción activada por defecto, no es imprescindible su escritura.
     
<Vis>MiPágina</Vis>   Abre el visor a pantalla completa con la página MiPágina cargada.
Se pueden cargar archivos Html, Gif, Jpg, Pdf, Txt, entre otros, tanto del disco local como de Internet o la propia red.
Para cerrar el visor basta con pulsar la tecla Esc
     
<File>MiPag.htm</File>  

Carga en pantalla el archivo MiPag.htm El archivo debe encontrarse en el mismo directorio que contiene la Ficha.
Al igual que en el comando anterior, se pueden cargar diferentes tipos de archivos.
     
<Mus>MiMus.wav</Mus>  

Hace sonar el archivo de música MiMus.wav Este archivo debe ser un Wav o Mid y encontrarse en el mismo directorio que contiene la Ficha
     
<Del>Informe</Del>  

Borra el Informe que lleva el nombre señalado y se encuentra en el mismo directorio que la aplicación. Este comando se puede utilizar para vaciar los informes antes de iniciar una nueva actividad. Sólo se puede ejecutar una vez desde cada Ficha.
Si deseamos borrar todos los informes que existan deberemos escribir:
   

<Del>*.*</Del>
     
<Run>MiProgram.exe</Run>  

Ejecuta el programa MiProgram.exe que se debe encontrar en el mismo directorio que contiene la Ficha.
Con este comando también se pueden abrir documentos que Windows reconozca por su extensión, sin necesidad de especificar a qué aplicación pertenecen.
Por ejemplo, podemos escribir:
 
   

<Run>xxx.doc</Run>

Y se nos abrírá el documento en el Word (Si se encuentra instalado correctamente en el ordenador este programa)
 

<Run>xxx.avi</Run>

Así se nos abrirá el reproductor multimedia visualizando el archivo.
   
   

Observe que de esta forma podemos abrir un mismo documento del tipo Html, por ejemplo, con un comando <File> o <Run> El primero nos lo abrirá en nuestro programa Corrector y el segundo en el navegador que tengamos instalado por defecto (Probablemente el Explorer)
   

El mismo comando <Run> nos puede permitir insertar actividades Clic dentro de nuestra Ficha.
   

<Run>ActClic.pcc</Run>

Si tenemos correctamente instalado el Clic (Versión 3.0), se nos abrirá con el archivo  ActClic.pcc cargado.
     

Opcionalmente se puede colocar el archivo a cargar en un subdirectorio de aquel que contiene la Ficha. En este caso deberíamos especificar el nombre de dicho subdirectorio dentro de los comandos <File> , <Mus> o <Run>
Así podríamos escribir:
  <File>Subdirect\MiPag.htm</File>
